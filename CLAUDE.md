# runner — R Package for Rolling Window Operations

## Build & Test
- Check package: `R CMD check .`
- Run tests: `Rscript -e 'tinytest::test_package("runner")'`
- Build docs: `Rscript -e 'roxygen2::roxygenize()'`
- Build pkgdown site: `Rscript -e 'pkgdown::build_site()'`
- Full build: `R CMD build .`

## Code Style
- R code is in `R/`, C++ in `src/` (Rcpp)
- Documentation: roxygen2 with markdown support
- Testing framework: tinytest (tests in `inst/tinytest/`)
- Follow existing naming conventions: snake_case for R functions

## Workflow
- Base branch is `main`
- Run R CMD check before submitting PRs
- Exported functions are declared in NAMESPACE (auto-generated by roxygen2)
- After modifying roxygen comments, regenerate docs with `roxygen2::roxygenize()`

## Architecture
- Core rolling window logic is in C++ (`src/runner.cpp`) via Rcpp
- `R/run.R` contains the main `runner()` S3 generic and methods
- `R/run_by.R` handles grouped operations
- Package supports data.frame, matrix, xts, and grouped_df inputs

## Protected Files
- NEVER modify `CLAUDE.md`, `.claude/`, or `.github/CODEOWNERS` — these are maintained by the project owner only

## Important
